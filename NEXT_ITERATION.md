# 下一迭代计划

## 迭代目标

拿到至少一家试点船司的 API 凭证后，完成真实 Track & Trace API 调用，并观察原始 JSON 返回字段。

## 需求范围

- 配置一家试点船司的 API 凭证。
- 使用只读库中的真实柜号调用官方 API。
- 打印完整原始 JSON 返回。
- 记录返回中是否包含事件、地点、船名航次、ETA。
- 如官方开通信息与当前默认地址或认证头不同，只调整对应 Provider。

## 不在本轮范围

- 不写公司数据库。
- 不保存本地 JSON 文件。
- 不做网页抓取。
- 不做可视化界面。
- 不解析、标准化或清洗 API 返回。
- 不扩展到非试点船司。

## 验收标准

- 至少一家船司能用真实凭证完成 API 请求。
- 控制台能打印原始 JSON。
- 缺少或错误凭证时能输出明确错误。
- `prod-db.yml` 不被修改、不被提交。
- 公司数据库只发生读取，不发生写入。

## 建议实现步骤

1. 申请并配置 Maersk、CMA CGM 或 MSC 任一船司的 API 凭证。
2. 按船司提供的信息设置环境变量。
3. 运行对应命令，使用只读库最新柜号验证 API 返回。
4. 如接口地址或认证头不匹配，只调整对应 Provider。
5. 确认原始 JSON 中可用于后续解析的字段。

## 需要确认的问题

- 第一家能拿到凭证的船司是哪一家？
- 船司给出的正式 API 地址和认证方式是什么？
- 是否需要在后续版本保存原始 JSON 到本地文件或数据库？
