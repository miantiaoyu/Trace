# 项目协作规则

本项目面向中文使用者。所有面向用户的界面、文案、说明文档和默认示例应优先使用中文，除非外部协议、代码标识符或第三方服务要求使用英文。

## 基本约束

- 所有文本文件、源码文件和配置文件统一使用 UTF-8 编码。
- 每次修改项目源码或文档前，必须先检查当前 Git 分支和工作区状态。
- 修改前应创建或切换到明确的任务分支；避免直接在主分支上开发。
- 修改完成后必须进行与改动范围匹配的验收；有测试、构建或静态检查时应运行相关检查。
- 验收通过后，再合并并推送到远程 GitHub 仓库。
- 按需阅读并维护项目状态文档、下一迭代需求文档和用户手册。

## 文档维护

- 开始实现前，如果任务与项目能力、范围或计划有关，先阅读 `PROJECT_STATUS.md` 和 `NEXT_ITERATION.md`。
- 实现完成后，更新 `PROJECT_STATUS.md`，只记录当前项目状态和当前能力。
- 实现完成后，更新 `NEXT_ITERATION.md`，只描述下一轮计划，不保留历史迭代流水。
- 当用户可见行为、安装方式、启动命令、使用流程或常见问题发生变化时，更新 `USER_MANUAL.md`。
- 状态文档和需求文档不保存过期历史、已完成迭代记录或重复内容。
